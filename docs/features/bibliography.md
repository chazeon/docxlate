# Bibliography & Citations

`docxlate` is specifically designed to handle complex citation and bibliography workflows by leveraging artifacts from real LaTeX/BibLaTeX runs.

## Why use LaTeX artifacts?

Most LaTeX to Word converters try to parse `.bib` files and re-implement citation styles. This often leads to subtle differences in citation order, numbering, and formatting compared to the original LaTeX document.

`docxlate` instead reads the files generated by your LaTeX compiler:
- **`.aux`**: Provides the mapping of citation keys to their appearance order and numeric IDs.
- **`.bbl`**: Contains the already-formatted bibliography entries as LaTeX code.
- **`.bcf`**: Provides additional citation metadata (used as a fallback).

## Citation Features

- **Numeric Range Compression**: Automatically compresses ranges like `[1, 2, 3]` to `[1-3]`. This is configurable via `plugins.bibliography.citation_compress_ranges` and `plugins.bibliography.citation_range_min_run`.
- **Hyperlinks**: Citations in the text are automatically hyperlinked to their corresponding entries in the bibliography.
- **Multiple Styles**: While numeric styles are the current focus, the engine is designed to be extensible to author-year styles.

## Bibliography Features

- **Consistent Rendering**: Bibliography entries are rendered using the content from the `.bbl` file, ensuring they match your LaTeX output exactly.
- **Jinja2 Templating**: You can customize how the bibliography section is structured and how individual entries are formatted using templates (see `plugins.bibliography.template`).
- **Automatic Indentation**: Controlled by the `plugins.bibliography.indent_in` configuration option.
